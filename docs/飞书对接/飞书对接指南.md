# 飞书（Feishu/Lark）对接指南

## 概述

飞书是字节跳动旗下的企业协作平台，本指南将帮助你配置 TinyClaw 与飞书的对接。

## 前提条件

- 一个飞书企业账号（或个人账号）
- 可以接收公网请求的服务器（或使用内网穿透工具如 ngrok）

## 创建飞书应用

### 1. 访问飞书开放平台

访问 [飞书开放平台](https://open.feishu.cn/app) 并登录。

### 2. 创建企业自建应用

1. 点击 "创建应用"
2. 选择 "企业自建应用"
3. 填写应用名称（例如 "TinyClaw AI Assistant"）
4. 选择应用头像和简介
5. 点击 "创建"

### 3. 获取凭证信息

在应用详情页的 **"凭证与基础信息"** 标签中：

1. 复制 **App ID**
2. 复制 **App Secret**
3. 复制 **Encrypt Key**（加密密钥）

将这些信息保存到 `.env` 文件中：

```bash
FEISHU_APP_ID=cli_xxx
FEISHU_APP_SECRET=xxx
FEISHU_ENCRYPT_KEY=xxx
```

## 配置权限

### 1. 配置权限

在应用详情页的 **"权限管理"** 标签中，添加以下权限：

- `im:resource`
- `im:message`
- `im:message:resource`

### 2. 配置事件订阅

在应用详情页的 **"事件订阅"** 标签中：

1. 点击 "配置事件订阅"
2. 填写 **接收事件服务器** 的 URL：
   ```
   http://your-server-ip:8080/webhook
   ```

   **注意**：如果是本地开发，需要使用内网穿透工具：
   - 使用 ngrok：`ngrok http 8080`
   - 使用 localtunnel：`lt --port 8080`

   然后使用生成的公网地址

3. 填写 **验证令牌**（Verification Token）：这个值会在事件订阅配置中生成
4. 勾选以下事件：
   - `im.message.receive_v1` (接收消息事件)

5. 点击 "保存"

### 3. 启用机器人

在应用详情页的 **"功能配置"** 标签中：

1. 开启 **机器人**
2. 填写机器人名称和描述
3. 上传机器人头像（可选）

## 启动飞书客户端

### 1. 安装依赖

```bash
cd tinyclaw
npm install
```

### 2. 配置环境变量

确保 `.env` 文件中包含飞书的凭证信息：

```bash
FEISHU_APP_ID=your_app_id
FEISHU_APP_SECRET=your_app_secret
FEISHU_ENCRYPT_KEY=your_encrypt_key
```

### 3. 编译代码

```bash
npm run build
```

### 4. 启动飞书客户端

```bash
npm run feishu
```

你应该看到类似以下的输出：

```
[2026-03-02T15:30:00.000Z] [INFO] Feishu webhook server listening on port 8080
[2026-03-02T15:30:00.000Z] [INFO] Configure your Feishu app to send events to: http://your-server:8080/webhook
```

### 5. 启动队列处理器

在另一个终端窗口中：

```bash
npm run queue
```

## 添加机器人到聊天

### 1. 安装应用

在飞书应用详情页：

1. 点击 **"版本管理与发布"**
2. 点击 **"开发版本"**
3. 点击 **"安装"**
4. 选择你的企业或测试账号
5. 完成安装

### 2. 添加机器人到聊天

在飞书客户端：

1. 打开一个私聊窗口
2. 搜索你的应用名称
3. 点击应用并发送第一条消息
4. 机器人会要求你配对（首次使用）

### 3. 配对用户

当有新用户第一次发送消息时，机器人会返回一个配对码。

在运行 TinyClaw 的机器上执行：

```bash
tinyclaw pairing approve <配对码>
```

## 测试

发送测试消息到飞书机器人的私聊窗口：

```
你好
```

你应该收到机器人的回复。

## 支持的功能

### 命令

- `/agent` - 查看可用的 Agent 列表
- `/team` - 查看可用的团队列表
- `/reset` - 重置对话历史

**使用示例**：

```
/reset @agent_id
```

### Agent 路由

你可以在消息开头使用 `@agent_id` 来指定特定的 Agent 处理消息：

```
@coding 请帮我写一个 Python 函数
```

### 团队路由

你可以在消息开头使用 `@team_id` 来指定特定的团队处理消息：

```
@dev 请帮我调试这段代码
```

## 配置说明

### 环境变量

| 变量名 | 说明 | 是否必需 |
|--------|------|----------|
| `FEISHU_APP_ID` | 飞书应用的 App ID | 是 |
| `FEISHU_APP_SECRET` | 飞书应用的 App Secret | 是 |
| `FEISHU_ENCRYPT_KEY` | 事件订阅的加密密钥 | 推荐 |
| `FEISHU_WEBHOOK_PORT` | Webhook 服务器端口（默认 8080） | 否 |

### 飞书服务器端口

飞书客户端启动一个 HTTP 服务器来接收飞书的 webhook 事件。默认端口是 8080，可以通过环境变量修改：

```bash
FEISHU_WEBHOOK_PORT=9000
```

然后在飞书开放平台配置相应的 webhook URL：

```
http://your-server:9000/webhook
```

## 高级配置

### 仅接收私聊消息

当前实现只处理 `p2p`（点对点）聊天，不处理群组消息。这是为了安全考虑。

如果你想启用群组消息，需要修改 `feishu-client.ts` 中的逻辑：

```typescript
// 当前只处理私聊
if (message.chat_type !== 'p2p') {
    return;
}

// 修改为处理私聊和群组
if (message.chat_type !== 'p2p' && message.chat_type !== 'group') {
    return;
}
```

### 消息类型支持

目前只支持文本消息（`text`）。未来可以扩展支持：

- 图片消息
- 文件消息
- 语音消息
- 视频消息

## 故障排查

### 1. "Invalid signature" 错误

- 检查 `FEISHU_ENCRYPT_KEY` 是否正确
- 确保飞书开放平台的事件订阅中使用的加密密钥与环境变量一致

### 2. "Failed to get access token" 错误

- 检查 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 是否正确
- 确保应用没有被禁用或删除

### 3. 收不到消息

- 检查 webhook URL 是否正确配置
- 检查服务器端口是否可访问
- 查看飞书开放平台的事件订阅日志
- 检查防火墙设置

### 4. URL 验证失败

- 确保 webhook URL 可以从公网访问
- 检查服务器是否正确响应 URL 验证请求
- 查看服务器日志中的 URL 验证请求

## 日志查看

飞书客户端的日志保存在：

```
.tinyclaw/logs/feishu.log
```

你可以实时查看日志：

```bash
tail -f .tinyclaw/logs/feishu.log
```

## 安全建议

1. **不要在公网暴露未加密的 webhook**：使用 HTTPS 或内网穿透工具的加密隧道
2. **定期更新凭证**：定期在飞书开放平台重新生成 App Secret
3. **启用签名验证**：确保 `FEISHU_ENCRYPT_KEY` 已配置
4. **限制访问来源**：在防火墙中只允许飞书服务器的 IP 访问 webhook 端口
