# 国产大模型配置示例

本文档提供智普（GLM）、Kimi（Moonshot）和 Qwen（通义千问）在 `settings.json` 中的配置示例。

## 配置示例

### 1. 智普（Zhipu）配置

```json
{
  "agents": {
    "zhipu-agent": {
      "name": "智普助手",
      "provider": "zhipu",
      "model": "glm-4",
      "working_directory": "/path/to/workspace/zhipu-agent"
    }
  }
}
```

**可用的 provider 值**：`zhipu` 或 `glm`

**推荐的模型**：
- `glm-4` - 通用场景
- `glm-4-plus` - 高质量生成
- `glm-4-air` - 快速响应
- `glm-4-flash` - 成本敏感

### 2. Kimi（Moonshot）配置

```json
{
  "agents": {
    "kimi-agent": {
      "name": "Kimi 助手",
      "provider": "kimi",
      "model": "kimi-k2.5",
      "working_directory": "/path/to/workspace/kimi-agent"
    }
  }
}
```

**可用的 provider 值**：`kimi` 或 `moonshot`

**推荐的模型**：
- `kimi-k2.5` - 最新版本（推荐）
- `kimi-k2.5-free` - 免费版本
- `moonshot-v1-128k` - 超长上下文

### 3. Qwen（通义千问）配置

```json
{
  "agents": {
    "qwen-agent": {
      "name": "通义千问助手",
      "provider": "qwen",
      "model": "qwen-max",
      "working_directory": "/path/to/workspace/qwen-agent"
    }
  }
}
```

**可用的 provider 值**：`qwen`、`tongyi` 或 `alibaba`

**推荐的模型**：
- `qwen-max` - 最高质量（推荐）
- `qwen-plus` - 平衡性能
- `qwen-turbo` - 快速响应
- `qwen2.5-72b-instruct` - 高性能大模型

### 4. 混合多个模型

```json
{
  "agents": {
    "fast-agent": {
      "name": "快速助手",
      "provider": "zhipu",
      "model": "glm-4-air",
      "working_directory": "/path/to/workspace/fast"
    },
    "quality-agent": {
      "name": "质量助手",
      "provider": "qwen",
      "model": "qwen-max",
      "working_directory": "/path/to/workspace/quality"
    },
    "research-agent": {
      "name": "研究助手",
      "provider": "kimi",
      "model": "kimi-k2.5",
      "working_directory": "/path/to/workspace/research"
    }
  }
}
```

### 5. 创建团队使用多个模型

```json
{
  "agents": {
    "zhipu-coder": {
      "name": "智普代码助手",
      "provider": "zhipu",
      "model": "glm-4",
      "working_directory": "/path/to/workspace/zhipu-coder"
    },
    "qwen-writer": {
      "name": "Qwen 文档助手",
      "provider": "qwen",
      "model": "qwen-max",
      "working_directory": "/path/to/workspace/qwen-writer"
    },
    "kimi-reviewer": {
      "name": "Kimi 审查助手",
      "provider": "kimi",
      "model": "kimi-k2.5",
      "working_directory": "/path/to/workspace/kimi-reviewer"
    }
  },
  "teams": {
    "dev-team": {
      "name": "开发团队",
      "agents": ["zhipu-coder", "qwen-writer", "kimi-reviewer"],
      "leader_agent": "zhipu-coder"
    }
  }
}
```

## 注意事项

1. **API 密钥配置**：使用这些模型前，需要安装 OpenCode CLI 并配置相应的 API 密钥
   ```bash
   # 安装 OpenCode
   npm install -g @opencode/cli

   # 配置 API 密钥
   opencode config set zhipu.api_key "your-zhipu-api-key"
   opencode config set kimi.api_key "your-moonshot-api-key"
   opencode config set qwen.api_key "your-dashscope-api-key"
   ```

2. **模型选择**：根据实际需求选择合适的模型，不同模型在性能、成本和能力上有所差异

3. **provider 别名**：可以使用别名简化配置
   - 智普：`zhipu` 或 `glm`
   - Kimi：`kimi` 或 `moonshot`
   - Qwen：`qwen`、`tongyi` 或 `alibaba`

4. **工作目录**：每个代理需要独立的工作目录，用于存储会话历史和配置

5. **使用方式**：在聊天中使用 `@agent_id` 前缀调用对应的代理
   ```
   @zhipu-fast 请帮我快速分析一下这个代码
   @qwen-main 请详细解释这个算法的原理
   @kimi-research 查找最新的 AI 研究论文
   ```
