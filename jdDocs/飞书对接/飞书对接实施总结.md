## 核心架构

### 消息流程

```
飞书用户 → 飞书服务器 → Webhook (HTTP POST)
          ↓
    feishu-client.ts
          ↓
    消息队列 (API)
          ↓
    queue-processor.ts
          ↓
    AI Agent 处理
          ↓
    队列响应
          ↓
    feishu-client.ts
          ↓
    飞书 API → 飞书用户
```

### 关键接口

**接收消息**:
- Webhook 端点: `POST /webhook`
- 事件类型: `im.message.receive_v1`
- 消息类型: `text`（目前）
- 聊天类型: `p2p`（私聊）

**发送消息**:
- API 端点: `POST /open-apis/im/v1/messages`
- 回复消息: `POST /open-apis/im/v1/messages/{message_id}/reply`
- 消息类型: `text`
- 接收者类型: `open_chat_id` 或 `open_id`

## 安全特性

1. **Webhook 签名验证**:
   - 使用 Encrypt Key 验证请求来源
   - 支持开发模式跳过验证

2. **访问令牌管理**:
   - 自动缓存和刷新
   - 5分钟缓冲期避免过期

3. **用户配对机制**:
   - 首次使用需要配对码
   - 防止未授权访问
