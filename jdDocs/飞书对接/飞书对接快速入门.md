# 飞书对接快速入门

## 5 分钟上手飞书机器人

### 第一步：创建飞书应用（2分钟）

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 点击 "创建应用" → "企业自建应用"
3. 填写应用名称（如 "TinyClaw AI"）
4. 记录以下信息：
   - **App ID**（如：`cli_a92a4d192838dcb2`）
   - **App Secret**（如：`YLD66hhnPAsWDRJGbOU3vbvLxgoXHoBH`）
   - **Encrypt Key**（加密密钥）

### 第二步：配置环境变量（1分钟）

在 `tinyclaw/.env` 文件中添加：

```bash
FEISHU_APP_ID=你的App_ID
FEISHU_APP_SECRET=你的App_Secret
FEISHU_ENCRYPT_KEY=你的Encrypt_Key
```

### 第三步：配置事件订阅（1分钟）

在飞书应用管理后台：

1. 进入 **"事件订阅"** 页面
2. 配置接收事件服务器：
   ```
   http://你的服务器IP:8080/webhook
   ```
   - 本地开发可以用 `ngrok http 8080` 生成公网地址
3. 勾选事件类型：`im.message.receive_v1`
4. 保存配置

### 第四步：启动服务（1分钟）

```bash
cd tinyclaw
npm run build      # 编译代码
npm run feishu     # 启动飞书客户端
```

在另一个终端：

```bash
npm run queue      # 启动队列处理器
```

### 第五步：测试机器人

1. 在飞书开放平台点击 **"安装"**
2. 在飞书客户端搜索你的应用
3. 发送第一条消息（需要配对）：
   ```bash
   tinyclaw pairing approve <配对码>
   ```
4. 再次发送消息，机器人应该会回复！

## 常用命令

- `/agent` - 查看可用的 AI Agent
- `/team` - 查看可用的团队
- `/reset @agent_id` - 重置特定 Agent 的对话

## 故障排除

### 收不到消息？

- 检查 webhook URL 是否正确
- 检查服务器端口 8080 是否可访问
- 查看 `.tinyclaw/logs/feishu.log` 日志文件

### 签名验证失败？

- 确认 `FEISHU_ENCRYPT_KEY` 与飞书后台一致
- 检查事件订阅配置是否保存成功
