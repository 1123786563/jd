## 🔧 故障排查

### 问题 1: 收不到飞书消息

**症状**: 飞书发送消息后没有任何日志

**检查**:
1. 确认 webhook URL 配置正确
   ```bash
   curl http://your-server:8080/webhook
   ```
   应该返回 "Not found"（说明服务器在运行）

2. 检查飞书开放平台的事件订阅日志
   - 进入 "事件订阅" → "推送日志"
   - 查看是否有失败记录

3. 检查服务器防火墙
   ```bash
   # macOS
   sudo pfctl -s all

   # Linux
   sudo iptables -L
   ```

**解决**: 确保 8080 端口可访问

### 问题 2: 签名验证失败

**症状**: 日志显示 "Invalid signature"

**原因**: `FEISHU_ENCRYPT_KEY` 不匹配

**解决**:
1. 检查 `.env` 文件中的 `FEISHU_ENCRYPT_KEY`
2. 确认与飞书开放平台的 "Encrypt Key" 一致
3. 重新启动飞书客户端

### 问题 3: 无法获取访问令牌

**症状**: 日志显示 "Failed to get access token"

**原因**: App ID 或 App Secret 错误

**解决**:
1. 检查 `.env` 文件中的凭证
2. 在飞书开放平台重新生成 App Secret
3. 更新 `.env` 文件
4. 重新启动服务

### 问题 4: 用户配对问题

**症状**: 首次使用需要配对，但没有收到配对码

**解决**:
1. 确保 `.tinyclaw/pairing.json` 存在
2. 查看飞书客户端日志中的配对码
3. 执行配对命令:
   ```bash
   tinyclaw pairing approve <配对码>
   ```

### 问题 5: 队列连接失败

**症状**: 日志显示 "fetch failed" 或 "connect ECONNREFUSED"

**原因**: 队列处理器未运行

**解决**:
1. 确保在另一个终端运行 `npm run queue`
2. 检查 `TINYCLAW_API_PORT` 环境变量（默认 3777）

## 🎯 最佳实践

### 1. 使用 HTTPS

生产环境建议使用 HTTPS:

```bash
# 使用 nginx 反向代理
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location /webhook {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

然后在飞书配置中使用 HTTPS URL:
```
https://your-domain.com/webhook
```

### 2. 配置健康检查

```bash
# 添加健康检查端点
curl http://localhost:8080/health
```

### 3. 设置自动重启

使用 pm2 管理进程:

```bash
npm install -g pm2

# 启动飞书客户端
pm2 start dist/channels/feishu-client.js --name feishu-bot

# 启动队列处理器
pm2 start dist/queue-processor.js --name queue-processor

# 设置开机自启
pm2 startup
pm2 save
```

### 4. 监控系统资源

```bash
# 查看进程状态
pm2 status

# 查看资源使用
pm2 monit

# 查看日志
pm2 logs feishu-bot
```
