# 在 TinyClaw 中使用智普、Kimi 和 Qwen 大模型

本文档介绍如何在 TinyClaw 项目中配置和使用智普（GLM）、Kimi（月之暗面）和 Qwen（通义千问）这三个国产大模型。

## 概述

TinyClaw 现已完整支持以下三个国产大模型：

- **智普（Zhipu / GLM）**：智谱 AI 推出的大语言模型系列
- **Kimi（Moonshot）**：月之暗面推出的多模态大模型
- **Qwen（通义千问）**：阿里巴巴推出的通义千问大模型系列

这些模型通过 OpenCode CLI 进行调用，需要安装 OpenCode 并配置相应的 API 密钥。

## 前提条件

### 1. 安装 OpenCode CLI

OpenCode 是一个统一的大模型调用工具，支持多个国产大模型。

```bash
# 安装 OpenCode CLI
npm install -g @opencode/cli

# 或使用其他包管理器
yarn global add @opencode/cli
pnpm add -g @opencode/cli
```

### 2. 获取 API 密钥

#### 智普（Zhipu）API 密钥

1. 访问 [智谱 AI 开放平台](https://open.bigmodel.cn/)
2. 注册并登录账号
3. 在控制台创建应用，获取 API Key
4. 配置环境变量：

```bash
export ZHIPU_API_KEY="your-zhipu-api-key"
```

#### Kimi（Moonshot）API 密钥

1. 访问 [Moonshot AI 开放平台](https://platform.moonshot.cn/)
2. 注册并登录账号
3. 在控制台创建应用，获取 API Key
4. 配置环境变量：

```bash
export MOONSHOT_API_KEY="your-moonshot-api-key"
```

#### Qwen（通义千问）API 密钥

1. 访问 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 注册并登录账号
3. 在控制台创建应用，获取 API Key
4. 配置环境变量：

```bash
export DASHSCOPE_API_KEY="your-dashscope-api-key"
```

### 3. 配置 OpenCode

运行以下命令配置各个模型的 API 密钥：

```bash
# 配置智普
opencode config set zhipu.api_key "your-zhipu-api-key"

# 配置 Kimi
opencode config set kimi.api_key "your-moonshot-api-key"

# 配置 Qwen
opencode config set qwen.api_key "your-dashscope-api-key"
```

## 配置 TinyClaw 代理

### 方法一：通过 TinyOffice Web 界面（推荐）

1. 启动 TinyClaw：

```bash
tinyclaw start
```

2. 启动 TinyOffice：

```bash
cd tinyclaw/tinyoffice
npm run dev
```

3. 在浏览器中打开 `http://localhost:3000`
4. 点击 "Agents" 菜单
5. 点击 "Add Agent" 按钮
6. 填写配置：
   - **Agent ID**：代理的唯一标识符（例如：zhipu-agent）
   - **Display Name**：显示名称（例如：智普助手）
   - **Provider**：选择对应的供应商
     - 智普：选择 `zhipu` 或 `glm`
     - Kimi：选择 `kimi` 或 `moonshot`
     - Qwen：选择 `qwen`、`tongyi` 或 `alibaba`
   - **Model**：输入模型名称（见下文模型列表）
   - **Working Directory**：可选，工作目录路径
   - **System Prompt**：可选，自定义系统提示词

7. 点击 "Create Agent" 保存

### 方法二：手动编辑配置文件

直接编辑 `~/.tinyclaw/settings.json`：

```json
{
  "agents": {
    "zhipu-assistant": {
      "name": "智普助手",
      "provider": "zhipu",
      "model": "glm-4",
      "working_directory": "/Users/me/tinyclaw-workspace/zhipu-assistant"
    },
    "kimi-assistant": {
      "name": "Kimi 助手",
      "provider": "kimi",
      "model": "kimi-k2.5",
      "working_directory": "/Users/me/tinyclaw-workspace/kimi-assistant"
    },
    "qwen-assistant": {
      "name": "通义千问助手",
      "provider": "qwen",
      "model": "qwen-max",
      "working_directory": "/Users/me/tinyclaw-workspace/qwen-assistant"
    }
  }
}
```

## 使用示例

### 1. 通过消息前缀调用

在任何聊天渠道（Discord、Telegram、WhatsApp）中：

```
@zhipu-assistant 请帮我写一个 Python 爬虫
@kimi-assistant 总结一下这篇文章的主要观点
@qwen-assistant 解释一下量子计算的基本原理
```

### 2. 通过 TinyOffice 聊天界面

1. 打开 `http://localhost:3000`
2. 点击左侧的 "Chat" 菜单
3. 选择对应的代理（Agent）
4. 输入消息并发送

### 3. 创建团队协作

可以将多个不同模型的代理组合成团队：

```json
{
  "teams": {
    "research-team": {
      "name": "研究团队",
      "agents": ["zhipu-assistant", "kimi-assistant", "qwen-assistant"],
      "leader_agent": "zhipu-assistant"
    }
  }
}
```

然后使用团队前缀：

```
@research-team 请分析一下这份报告并给出建议
```

## 性能对比和选择建议

### 成本对比

| 模型供应商 | 相对成本 | 说明 |
|-----------|---------|------|
| **智普（GLM）** | 低-中 | 价格适中，性价比高 |
| **Kimi（Moonshot）** | 中-高 | 免费版可用，付费版性能更好 |
| **Qwen（通义千问）** | 低-中 | 阿里云定价，灵活计费 |

### 性能特点

#### 智普（GLM）
- ✅ 中文理解能力强
- ✅ 代码生成质量高
- ✅ 响应速度快
- ⚠️ 英文能力相对较弱

#### Kimi（Moonshot）
- ✅ 超长上下文支持（128K）
- ✅ 文档分析能力强
- ✅ 多模态支持
- ⚠️ 成本较高

#### Qwen（通义千问）
- ✅ 中英文均衡
- ✅ 生态完善（阿里云）
- ✅ 多种规模可选
- ✅ 代码能力优秀
- ⚠️ 需要阿里云账号

### 使用场景推荐

| 场景 | 推荐模型 |
|-----|---------|
| 代码生成 | `glm-4`、`qwen-max` |
| 文档总结 | `kimi-k2.5`、`qwen-long` |
| 中文对话 | `glm-4`、`qwen-max` |

| 快速响应 | `glm-4-air`、`qwen-turbo` |

## 故障排查

### 1. API 密钥错误

**错误信息**：
```
API authentication failed
```

**解决方案**：
```bash
# 重新配置 API 密钥
opencode config set zhipu.api_key "your-new-api-key"
```

### 2. 模型名称错误

**错误信息**：
```
Model not found
```

**解决方案**：
检查模型名称是否正确，参考上面的模型列表。

### 3. 配额不足

**错误信息**：
```
Quota exceeded
```

**解决方案**：
- 检查账户余额
- 升级套餐
- 切换到免费版本或更便宜的模型

### 4. 网络连接问题

**错误信息**：
```
Connection timeout
```

**解决方案**：
- 检查网络连接
- 设置代理（如果需要）
- 重试请求

## 最佳实践

### 1. 混合使用多个模型

针对不同任务使用最适合的模型：

```json
{
  "agents": {
    "fast-agent": {
      "name": "快速助手",
      "provider": "zhipu",
      "model": "glm-4-air",
      "working_directory": "/Users/me/tinyclaw-workspace/fast"
    },
    "quality-agent": {
      "name": "质量助手",
      "provider": "qwen",
      "model": "qwen-max",
      "working_directory": "/Users/me/tinyclaw-workspace/quality"
    },
    "research-agent": {
      "name": "研究助手",
      "provider": "kimi",
      "model": "kimi-k2.5",
      "working_directory": "/Users/me/tinyclaw-workspace/research"
    }
  }
}
```

### 2. 使用团队协作

将不同模型的代理组成团队，发挥各自优势：

```
@research-team 请帮我分析一下这个技术方案：
1. @fast-agent 先给出初步想法
2. @quality-agent 深入分析细节
3. @research-agent 提供参考文献
```

### 3. 配置系统提示词

为不同模型配置专门的系统提示词：

```json
{
  "agents": {
    "zhipu-coder": {
      "name": "代码生成器",
      "provider": "zhipu",
      "model": "glm-4",
      "system_prompt": "你是一个专业的代码生成助手，擅长生成高质量的 Python 和 TypeScript 代码。请确保代码符合最佳实践，包含必要的注释。"
    }
  }
}
```
